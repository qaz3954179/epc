## 二、小程序整体信息架构

### 核心决策

> 小程序不适合复杂的管理后台。**孩子端是核心体验，家长端做轻量版。**

```
小程序定位：
- 孩子：随时打卡、看进度、兑奖品（移动端高频场景）
- 家长：快速查看孩子数据、审批/回应、简单管理（移动端低频场景）
- 复杂管理（任务创建、奖品管理、SDI分析）仍留在 Web 端
```

### Tab 结构设计

**方案：角色切换制**

```
小程序底部 Tab 根据当前角色动态变化：

┌─────────────────────────────────────────────────────┐
│                  孩子端 Tab                          │
├──────────┬──────────┬──────────┬────────────────────┤
│  🏠 首页  │  📝 任务  │  🛒 商城  │  👤 我的         │
│  学习币   │  打卡     │  奖品    │  成就+设置       │
│  进度     │  考试     │  兑换    │  成长            │
└──────────┴──────────┴──────────┴────────────────────┘

┌─────────────────────────────────────────────────────┐
│                  家长端 Tab                          │
├──────────┬──────────┬──────────┬────────────────────┤
│  🏠 花园  │  📊 数据  │  ⚡ 审批  │  👤 我的         │
│  宝贝概览 │  监控面板 │  愿望回应 │  设置+宝贝管理   │
│  星球列表 │  任务记录 │  考试安排 │  推广            │
│          │  学习币   │          │                  │
└──────────┴──────────┴──────────┴────────────────────┘

切换方式：
- 登录时选择角色（孩子/家长），或
- 在"我的"页面切换身份
- 一个微信账号可以关联多个角色（家长 + 孩子的宝贝）
```

## 四、交互设计原则

### 孩子端设计原则

1. **大按钮、大字体**
   - 按钮高度 ≥ 60px
   - 主文字 ≥ 16px，标题 ≥ 20px
   - 触摸目标 ≥ 44px × 44px

2. **色彩鲜明**
   - 主色：渐变紫 → 粉（#667eea → #764ba2）
   - 成功：绿色（#43e97b → #38f9d7）
   - 奖励：橙色（#f093fb → #f5576c）
   - 与现有 Web 端保持一致

3. **即时反馈**
   - 任何操作都要有视觉反馈
   - 打卡成功 → 弹窗 + 震动 + 音效
   - 加载时 → 有趣的 loading 动画

4. **语言拟人化**
   - "太棒了！完成 +10 🪙"
   - "你的星球在等你哦~"
   - 不用"操作成功"这种机器语言

5. **简化流程**
   - 打卡 → 一步到位，不需要确认
   - 兑换 → 弹窗确认后直接扣币
   - 考试 → 进入后直接开始

### 家长端设计原则

1. **信息密度适中**
   - 比孩子端更紧凑，但不像 Web 端那样密集
   - 关键数据一眼看到

2. **操作便捷**
   - 愿望回应 → 一键操作（种种子/放一放）
   - 数据查看 → 快速切换宝贝

3. **角色切换流畅**
   - 家长可以在小程序内切换到孩子视角
   - 方便家长预览孩子看到的界面

## 六、登录与认证流程

```
┌─────────────────────────────────────┐
│          小程序启动                   │
├─────────────────────────────────────┤
│                                     │
│  1. wx.login() → 获取 code          │
│  2. 发送 code → 后端                │
│  3. 后端用 code 换微信 openId       │
│  4. 匹配逻辑：                       │
│     - openId 已绑定用户 → 登录成功   │
│     - openId 未绑定 →               │
│       - 有邀请码/手机号 → 绑定登录   │
│       - 新用户 → 注册流程            │
│  5. 返回 JWT → 小程序存储           │
│  6. 用户角色判断：                   │
│     - parent → 家长端               │
│     - child → 孩子端                │
│     - 两者都有 → 选择角色            │
│                                     │
└─────────────────────────────────────┘

Casdoor 整合方案：
- 方案 A：后端直接处理微信登录（绕过 Casdoor）
- 方案 B：Casdoor 支持微信 OAuth（需要确认）
- 推荐：方案 A 更简单，小程序场景不需要 SSO
```
