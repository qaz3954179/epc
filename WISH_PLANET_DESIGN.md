# 🪐 愿望星球 — 产品设计文档

> 每个愿望都是一颗正在诞生的星球。

---

## 核心理念

不是"许愿 → 分配任务 → 兑换奖品"的交易逻辑，而是让孩子通过愿望学会：

1. **认知现实** — 了解愿望背后需要什么
2. **理解代价** — 任何美好都需要付出和等待
3. **学会规划** — 大愿望拆成小步骤
4. **亲子对话** — 家长和孩子有意义的沟通

---

## 状态机

```
许愿(wishing) ── 家长回应 ──→ 萌芽(sprouting)
                                     │
                              完成知识卡片
                                     │
                              展开冒险地图
                                     │
                            ┌─── 旅程中(journey) ───┐
                            │     (可开新星球)       │
                            │                        │
                       完成所有里程碑               │
                            │                        │
                         诞生(born)                  │
                       (可再开新星球)                │
                            │                        │
                      永久存在，                      │
                      可回顾                        │

家长搁置 ──→ 休眠(shelved)
              (不占用星球名额)
```

## 知识卡片（AI 自动生成）

### 生成时机
孩子写完愿望 → 家长"种下种子"后 → 系统自动生成 5 张卡片。

### 卡片类型

| 类型 | 图标 | 目的 | 示例 |
|------|------|------|------|
| 知识卡 | 📖 | 事实认知 | 仓鼠的寿命、习性、食物 |
| 思考卡 | 🤔 | 思辨能力 | "仓鼠只能活2-3年意味着什么？" |
| 经济卡 | 💰 | 财商启蒙 | 养仓鼠的完整费用清单 |
| 情感卡 | ❤️ | 共情能力 | 仓鼠生病了该怎么办 |
| 拓展卡 | 🌍 | 知识广度 | 全世界有多少种仓鼠 |

### 交互
- 每完成一张卡片，星球表面多一层纹理（大气层 → 水 → 绿色 → 生命）
- 答对测验题 → 星球闪光 ✨
- 家长可看到孩子的答题数据（特别是开放性回答）

---

## 里程碑（4 种类型）

### 🌿 习惯养成（habit）
- 证明"我能坚持"
- 示例：连续7天给绿植浇水
- 验证方式：绑定现有任务系统（Items），每天打卡

### 📝 知识考试（knowledge）
- 证明"我了解了"
- 示例：通过"仓鼠知识测试"（80%以上）
- 验证方式：复用现有考试系统（Exams）

### 💰 攒币（coins）
- 证明"我能计划和等待"
- 示例：攒够200学习币买仓鼠用品
- 验证方式：直接读取 coins 余额

### 🌍 体验实践（experience）
- 证明"我做到了"
- 示例：给仓鼠笼子打扫卫生
- 验证方式：家长手动确认（拍照/点击"已验证"）

### 解锁逻辑

```
知识卡片全部读完 ──→ 解锁冒险地图
                            │
                    ┌───────┼───────┐
                    │       │       │
               🌿 习惯   💰 攒币   📝 考试
               (可同时) (可同时)  (需先完成2个习惯)
                    │       │       │
                    └───────┼───────┘
                            │
                    🌍 体验实践
               (需前3类各完成1个)
                            │
                    🎆 星球诞生
```

---

## 星球数量限制

| 条件 | 可同时活跃的星球数 |
|------|-------------------|
| 初始 | 1 颗 |
| 第1颗星球进入"旅程中"（至少1个里程碑完成） | 解锁第2颗 |
| 有1颗星球已诞生 ✨ | 解锁第3颗 |
| 上限 | **3 颗** |

休眠(shelved)状态的星球**不占用名额**。

## 星球搁置 → 慢慢变暗

### 视觉变化时间线

| 无操作天数 | 状态 | 视觉 |
|-----------|------|------|
| 0-7天 | 正常 | 明亮，有光晕，缓慢旋转 |
| 7天 | 警告 | 光晕变淡，颜色略微暗淡 |
| 14天 | 暗淡 | 星球变灰，纹理模糊，出现小乌云 ☁️ |
| 21天 | 沉睡 | 几乎看不见，"这个愿望在等待你的回应..." |
| 30天 | 冬眠 | 变成小石头飘在角落，**保留不消失** |

### 唤回机制

**孩子端：**
```
🔔 你的仓鼠星球好像有点寂寞了……去看看它吧？
```

**家长端：**
```
🔔 宝贝的"养仓鼠"星球已经 7 天没有进展了
   [ 💬 鼓励 ]  [ 📝 调整难度 ]  [ 💤 先休息 ]
```

### 唤回后恢复

| 唤回时间 | 恢复亮度 |
|---------|---------|
| 7天内 | 100% |
| 14天内 | 70% |
| 21天+ | 50%（需几次互动才能完全恢复） |

## 与现有系统的衔接

| 现有系统 | 融入方式 |
|---------|---------|
| Items（任务） | 里程碑绑定日常任务打卡 |
| Coins（学习币） | 里程碑设置攒币目标，赚币推动星球成长 |
| Exams（考试） | "守护者考试"复用现有考试功能 |
| Achievements（成就） | 星球诞生解锁专属成就 |
| Prizes（奖品商城） | 实物愿望可关联商城商品 |

## 数据模型（初步）

```python
class WishStatus(str, Enum):
    wishing = "wishing"       # 等待家长回应
    sprouting = "sprouting"   # 知识探索中
    journey = "journey"       # 冒险进行中
    born = "born"             # 星球诞生
    shelved = "shelved"       # 休眠/搁置


class MilestoneType(str, Enum):
    habit = "habit"           # 习惯养成
    knowledge = "knowledge"   # 知识考试
    coins = "coins"           # 攒币
    experience = "experience" # 体验实践


class KnowledgeCard(SQLModel):
    """知识卡片"""
    id: UUID
    planet_id: UUID
    card_type: str           # knowledge / thinking / economic / emotional / expand
    title: str
    content: dict            # 文本、选择题、开放题等
    order: int               # 阅读顺序
    is_read: bool = False
    child_answer: dict | None  # 孩子的回答
    completed_at: datetime | None


class Milestone(SQLModel):
    """冒险里程碑"""
    id: UUID
    planet_id: UUID
    milestone_type: MilestoneType
    title: str
    description: str
    requirements: dict       # 具体完成条件
    reward_coins: int = 0    # 学习币奖励
    reward_effect: str       # 星球视觉效果描述
    is_completed: bool = False
    completed_at: datetime | None
    # 绑定关系
    bound_item_id: UUID | None    # 关联的任务
    bound_exam_id: UUID | None    # 关联的考试
    coin_target: int | None       # 攒币目标


class WishPlanet(SQLModel, table=True):
    """愿望星球"""
    id: UUID
    user_id: UUID           # 宝贝
    title: str
    description: str
    emoji: str
    planet_color: str       # 星球颜色
    
    status: WishStatus
    growth_percent: float   # 0-100
    
    # 家长回应
    parent_message: str | None
    parent_reason: str | None  # 暂缓原因
    
    # 内容（JSON 数组，存储序列化的卡片和里程碑）
    knowledge_cards: list[dict]
    milestones: list[dict]
    
    # 进度
    cards_read: int = 0
    milestones_completed: int = 0
    
    # 时间
    wished_at: datetime
    sprouted_at: datetime | None
    born_at: datetime | None
    last_active_at: datetime  # 最后活跃时间（用于计算变暗）
    
    # 证书
    certificate_data: dict | None
```

## 前端页面规划

### 孩子端
1. **星图首页** (`/wishes`) — 查看所有星球的星图视图
2. **星球详情页** (`/wishes/:id`) — 单颗星球的详细信息、进度、卡片
3. **知识卡片阅读页** — 沉浸式卡片阅读+答题
4. **冒险地图页** — 可视化里程碑进度

### 家长端
1. **星球花园** (`/parent/wishes`) — 所有孩子的星球概览
2. **愿望审核页** — 新愿望通知 + 回应
3. **星球管理页** (`/parent/wishes/:id`) — 调整里程碑、发鼓励、查看孩子回答
4. **证书查看页** — 查看已诞生的星球证书

## 实现阶段规划

### Phase 1：MVP
- 后端数据模型 + API
- 孩子端：许愿、查看星球列表、阅读知识卡片
- 家长端：回应愿望（种种子/放一放）、查看星球状态
- AI 知识卡片生成接口

### Phase 2：冒险地图
- 里程碑系统（4种类型）
- 与现有 Items/Coins/Exams 系统的对接
- 星球成长视觉效果
- 家长管理里程碑

### Phase 3：完整体验
- 星球变暗/唤回机制
- 星球数量限制逻辑
- 星球诞生动画 + 证书生成
- 成就系统联动
